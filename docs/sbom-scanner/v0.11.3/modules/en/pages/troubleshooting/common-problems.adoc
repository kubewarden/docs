This page documents common errors users may encounter while running {sboms-name}, and how to address them.

== Docker Hub pull rate limit reached

=== Problem

Scans end up in an error state with a message similar to:

[source,text]
----
You have reached your unauthenticated pull rate limit
----

This happens because anonymous image pulls from https://hub.docker.com/[Docker Hub] (`docker.io`) are subject to rate limiting. Once the limit is hit, further pulls are rejected until the window resets.

See the official Docker documentation for full details:

* https://docs.docker.com/docker-hub/usage/[Docker Hub usage and limits]
* https://docs.docker.com/docker-hub/usage/pulls/[Docker Hub pull rate limits]
* https://www.docker.com/increase-rate-limits/[Download rate limit overview]

=== Solution

Authenticate against Docker Hub so that pulls count against your account's higher limits, instead of the shared anonymous quota. Refer to Docker's documentation for the available options:

* https://docs.docker.com/docker-hub/usage/#authenticate-to-increase-your-pull-rate-limit[Authenticate to increase your pull rate limit]
* https://docs.docker.com/subscription/[Subscriptions and higher limits]

Once you have credentials, configure {sboms-name} to use them by creating a `kubernetes.io/dockerconfigjson` Secret in the {sboms-name} installation namespace and referencing it from the relevant `Registry` resource (or from the `authSecret` field of a `WorkloadScanConfiguration` to propagate the credentials to all managed registries). The full procedure — including the exact `kubectl create secret docker-registry` invocation and the `Registry` spec — is documented in xref:user-guide/private-registries.adoc[Private Registries].
